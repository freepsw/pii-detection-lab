# 거버넌스·마스킹 (Governance) — 탐지 다음: 보호·분류·증명

> **이 문서는 언제 읽나요?** `80_governance` 노트북과 함께 — 탐지 결과를 실제로 보호·증명할 때.  
> · **선행:** [`04_결과_해석과_단계_선택.md`](04_결과_해석과_단계_선택.md) · **다음:** [`06_내_데이터에_적용.md`](06_내_데이터에_적용.md) · **소요:** 10분

탐지(S1~S5)로 끝나면 데이터는 **여전히 노출**돼 있습니다. 거버넌스는 탐지 결과로 데이터를 **보호(마스킹)**
하고 **분류(태그)** 하며 그 상태를 **증명(검증)** 하는 마지막 한 바퀴입니다. 이 문서는 `80_governance/`
노트북이 무엇을·왜 하는지 안내합니다(코드는 노트북에, 여기서는 개념·선택·검증법).

> 선행 개념: [`02_핵심_개념.md`](02_핵심_개념.md) ⑪ 마스킹·⑫ 거버넌스. 트랙 구분: [`04_결과_해석과_단계_선택.md`](04_결과_해석과_단계_선택.md) §3 직관 ④.

## 1. 왜 거버넌스인가 — 탐지 → 보호 → 증명 폐곡선

탐지는 "어디에 PII가 있나"를 알려줄 뿐입니다. 실제로 안전해지려면 ① 가리고(마스킹) ② 분류하고(태그)
③ 정말 가려졌는지 증명해야 합니다. 이 폐곡선이 [`03_단계별_아키텍처.md`](03_단계별_아키텍처.md) 도식 (e)입니다.

## 2. 두 가지 마스킹

| 방식 | 대상 | 메커니즘 | 특징 |
|---|---|---|---|
| **컬럼 마스크** | 정형 컬럼 전체 | Unity Catalog `SET MASK` + UDF 7종 | *조회 시점* 권한 분기(`is_account_group_member('pii_full_access')`) — 권한자는 원본, 비권한자는 마스킹. 형식 불일치 시 기본 마스크(fail-closed) |
| **텍스트 부분 마스킹** | 자유텍스트 안 span | `type_map` 토큰으로 치환(`홍길동`→`[이름]`) | offset 안전을 위해 뒤에서 앞으로(start 내림차순) 치환. `text_corpus_masked`·`span_mask_audit` 산출 |

## 3. 어느 단계로 마스킹할까 — 두 verdict ★

컬럼과 텍스트는 **다른 트랙**이라, 각 트랙에서 가장 정확한 단계를 씁니다(`80_masking_governance.py` 의
`REC_STAGE`·`REC_SPAN_STAGE` 상수):

- **컬럼 태깅·마스크 → S2** (컬럼 F1 최고 0.9091)
- **텍스트 span 마스킹 → S4** (span F1 최고 0.9838)

왜 다른가? 컬럼 판정은 "이 칸에 PII가 있나/없나"만 보므로 S3·S4가 같은 결과(col F1 0.84 동률)지만, 텍스트
부분 마스킹은 **정확한 경계**가 중요해 파인튜닝 S4(0.9838)가 가장 안전합니다. (자세히: RESULTS §3 직관 ④.)

## 4. "잔존 PII 0건" 검증 — 마스킹이 됐다는 측정 가능한 증명

마스킹의 핵심 보증은 **마스킹 후 텍스트에 gold PII가 하나도 남지 않는 것**입니다. 90_eval의 C6과
80_governance의 검증 셀이 동일 쿼리로 이를 측정합니다:

```sql
SELECT count(*) FROM <카탈로그.스키마>.text_corpus_masked m
JOIN <카탈로그.스키마>.span_ground_truth g ON m.doc_id = g.doc_id
WHERE locate(g.pii_value, m.masked) > 0
  AND (length(g.pii_value) >= 4
       OR (g.entity_type IN ('PERSON','ADDRESS') AND length(g.pii_value) >= 2));  -- 0이 이상적
```

`length>=4` 만 쓰면 **한국 이름(대부분 3자)** 을 전부 건너뛰는 사각지대가 생기므로 PERSON/ADDRESS는
2자부터 검사합니다. 결과가 0이 아니면 노트북이 `assert` 로 실패해 **미마스킹 PII 유출을 차단**합니다(fail-closed).
내 데이터에 적용할 때도 이 쿼리로 "잔존 0건"을 직접 확인하세요.

## 5. 태그 vs 마스크 (혼동 주의)

`SET TAGS`(분류 메타데이터)는 컬럼에 보호등급·카테고리 라벨을 붙일 뿐 **그 자체로 가리지는 않습니다**.
실제 가림은 별도의 `SET MASK`(⑪)입니다. 태그 기반 ABAC(속성기반 접근제어) 정책으로의 승격은 확장 로드맵입니다.
