## ▶️ 노트북 실행 순서

위 **읽기 순서**와 별개로, 노트북은 **폴더 번호 순서**대로 실행합니다. **`00_setup/00_config` 를
가장 먼저 한 번 실행**하면(카탈로그·스키마·Volume·백엔드 설정), 이후 각 노트북 첫 셀에서 이
설정을 자동 상속합니다.

> ⚠️ **반드시 `03_pii_detection_lab` 폴더 전체를 import하세요** (Workspace > Import, zip/.dbc 디렉터리 단위). **단일 노트북만 import하면 실행이 깨집니다** — `00_config`가 실행 중 노트북 경로 기준으로 `_common`을, `01_generate_data`가 `data/`를 찾기 때문입니다.

> **1분 사전점검(권장)** — 긴 실습 전에 [`../../00_setup/00_preflight`](../../00_setup/00_preflight) 를 Run All로 돌려 클러스터·카탈로그·LLM 엔드포인트·`data/` 존재를 점검합니다(읽기전용·비차단). 전부 PASS면 진행, 경고가 있으면 [`01_환경_설정.md`](01_환경_설정.md) 해당 항목을 확인하세요.

| 순서 | 폴더 | 단계 | 하는 일 |
|---|---|---|---|
| 0 | `00_setup/` | 설정 | 카탈로그·스키마·Volume 생성, 백엔드·임계값 설정, 데이터 적재 |
| 1 | `10_S1_pattern/` | **S1** | 정규식으로 정형 PII를 컬럼·span 단위로 탐지 |
| 2 | `20_S2_pattern_llm/` | **S2** | 패턴 결과에 LLM을 더해 비정형 PII 회수 + 오탐 제거 |
| 3 | `30_S3_pattern_ner_llm/` | **S3** | 사전학습 NER + 저신뢰 구간만 LLM으로 보내는 cascade |
| 4 | `40_S4_pattern_nerft_llm/` | **S4** | **도메인 파인튜닝 NER** + cascade (정확도·비용 동시 최적화) — 노트북 2개: `S4a_generate_train_corpus` → `S4b_ner_finetune_cascade` |
| 5 | `50_S5_llm_only/` | **S5** | LLM 단독 탐지 (가장 단순한 구성) |
| 6 | `80_governance/` | 거버넌스 | 탐지 결과로 UC 태그·컬럼 마스크·텍스트 부분 마스킹 적용 |
| 7 | `90_eval/` | 평가·검증 | 5단계 비교 그리드 집계 + 독립 검증(10/10 PASS) |
| 8 | `60_monitor/` | **Day-2 운영(선택)** | 라벨없는 드리프트 탐지·규칙 자동교정(60a) + NER 챔피언/챌린저 재학습·승격 게이트(60b) |

> **문서 번호(읽기 순서)와 노트북 폴더 번호(실행 순서)는 다를 수 있습니다** — 예: 거버넌스 문서는 `05_거버넌스_마스킹.md`(읽기 5번)이지만, `80_governance` 노트북은 `90_eval` 전에(실행 6번) 돌립니다.

## 🗺️ 노트북 한눈 지도

노트북을 **열기 전에** "지금 이 폴더가 뭐고, 무엇을 넣어 무엇을 만들고, 잘 됐는지 어떻게 아는지"를

| 폴더 | 단계 | 한 줄 설명 | 핵심 입력 | 핵심 출력 | 기대 헤드라인 (span exact F1 / 라우팅 / 비용·1k) | 다음 |
|---|---|---|---|---|---|---|
| `00_setup/` | 설정·적재 | 위젯 4개로 카탈로그·백엔드 설정 → CSV를 Delta로 적재 | `data/*.csv` | 9개 테이블 + `text_corpus`(약 4,860 docs) | — | S1 |
| `10_S1_pattern/` | **S1** | 정규식만 — 정형 PII는 잡지만 **이름·주소 0%** | `text_corpus` | `col_predictions`·`span_predictions`·`span_coverage` | **0.73** / — / **$0** | S2 |
| `20_S2_pattern_llm/` | **S2** | 패턴 + **LLM 전수** — 비정형 회수·오탐 제거 | `text_corpus`, S1 | `llm_col_raw`·`span_llm_raw`·`*_predictions(S2)` | **0.9448** / 100% / **$0.132** | S3 |
| `30_S3_pattern_ner_llm/` | **S3** | 패턴 + **사전학습 NER** + cascade(저신뢰만 LLM) | `text_corpus` | `*_predictions(S3)` | **0.8966** / 10.9% / **$0.0244** | S4 |
| `40_S4_pattern_nerft_llm/` | **S4** | 패턴 + **파인튜닝 NER** + cascade — **최적**(정확↑·비용↓) | `text_corpus`(+ 학습 코퍼스) | 파인튜닝 모델 · `*_predictions(S4)` | **0.9838** / **0%** / **$0.01** | S5 |

> **`40_S4`는 노트북 2개**: `S4a_generate_train_corpus`(평가와 분리된 학습 코퍼스, seed=777) → `S4b_ner_finetune_cascade`(파인튜닝 + cascade). 먼저 S4a, 다음 S4b.

> 수치는 **합성 데이터 상한치**(gpt-oss-120b · 4,860 docs · span exact F1, README §2 기준). 실데이터 기대치·읽는 법은 [`04_결과_해석과_단계_선택.md`](04_결과_해석과_단계_선택.md), 전환은 [`06_내_데이터에_적용.md`](06_내_데이터에_적용.md).
